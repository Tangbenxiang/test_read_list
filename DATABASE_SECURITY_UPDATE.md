# 数据库安全规则更新指南

## 问题描述
当前 `books` 集合的权限设置为"所有用户可读，仅创建者可写"，这导致所有用户都可以修改自己创建的书籍状态。为了确保只有管理员可以修改书籍状态，需要更新数据库安全规则。

## 解决方案
更新 `books` 集合的安全规则，确保只有管理员可以写入数据。

## 操作步骤

### 1. 登录微信云开发控制台
1. 打开微信开发者工具
2. 点击"云开发"按钮，进入云开发控制台
3. 选择正确的环境（如：`cloudbase-4gnknimqbe0440c9`）

### 2. 更新 books 集合安全规则
1. 在左侧导航栏点击"数据库"
2. 找到 `books` 集合，点击集合名称
3. 点击"权限设置"标签页
4. 点击"自定义规则"按钮

### 3. 设置新的安全规则
将现有规则替换为以下内容：

```javascript
{
  "read": true,  // 所有用户可读
  "write": "get('database.admins').where({openid: auth.openid, role: 'admin'}).count() > 0"
}
```

**规则说明：**
- `read: true`：所有用户都可以读取书籍数据
- `write` 规则：只有当前用户（`auth.openid`）在 `admins` 集合中存在且 `role` 为 `admin` 时，才可以写入数据

### 4. 验证规则语法
1. 点击"校验规则"按钮检查语法是否正确
2. 如果语法正确，点击"确定"保存规则

### 5. 测试权限控制
1. 使用管理员账号测试：应该可以正常修改书籍状态
2. 使用普通用户账号测试：尝试修改书籍状态应该会显示权限错误

## 其他集合权限建议

### admins 集合
建议保持"仅管理员可读写"的权限设置，确保只有现有管理员可以添加新的管理员。

### feedback 集合
当前设置为"所有用户可写，仅管理员可读"，这符合需求，无需更改。

## 注意事项

1. **权限变更影响**：更新规则后，非管理员用户将无法修改任何书籍数据（包括状态）
2. **数据一致性**：确保 `admins` 集合中已正确配置了管理员用户
3. **测试验证**：务必进行充分测试，确保权限控制正常工作
4. **备份规则**：在修改前建议记录原有规则，以便需要时恢复

## 故障排除

### 问题1：规则保存失败
**可能原因**：语法错误
**解决方案**：检查规则语法，确保JSON格式正确

### 问题2：管理员无法写入
**可能原因**：
1. `admins` 集合中没有当前用户的记录
2. `openid` 或 `role` 字段不匹配
**解决方案**：检查 `admins` 集合，确保管理员用户的 `openid` 和 `role` 字段正确

### 问题3：普通用户错误地获得了写入权限
**可能原因**：规则配置错误
**解决方案**：重新检查规则，确保 `write` 条件正确验证管理员身份
