# 项目协作提示词模板

## 📋 模板使用说明

本模板为儿童阅读记录微信小程序项目的标准协作流程，确保每次会话高效、结构清晰且记录完整。

---

## 1. 会话记录机制

### 文件命名规范
每次会话开始时，自动创建一个以系统时间命名的会话记录文件：
```
SESSION_SUMMARY_YYYYMMDD_HHMM.md
```
示例：`SESSION_SUMMARY_20260304_1430.md`

### 记录内容结构
```markdown
# 会话记录：{任务简述} (YYYY-MM-DD HH:MM)

## 📌 当前任务
- {简要描述本次会话的主要目标}

## 🔍 项目状态快照
- **Git分支**: {当前分支}
- **最近提交**: {最近提交信息}
- **修改文件**: {列出已修改的文件}

## 📝 实施步骤
1. {步骤1描述}
2. {步骤2描述}
3. {步骤3描述}

## ✅ 已完成的工作
- [ ] {任务1}
- [ ] {任务2}

## 🎯 下一步计划
- {下一步1}
- {下一步2}

## 📁 重要文件参考
- [文件名1](文件路径1) - {文件用途}
- [文件名2](文件路径2) - {文件用途}

## ⚠️ 注意事项/问题
- {需要注意的问题或限制}

---

*记录时间：YYYY-MM-DD HH:MM*
*会话状态：进行中/已完成*
```

### 自动化脚本建议
在 `.claude/settings.local.json` 中添加钩子，自动创建会话文件：
```json
{
  "hooks": {
    "before_tool_call": "echo '# 会话记录：'$(date '+%Y%m%d_%H%M') > SESSION_SUMMARY_$(date '+%Y%m%d_%H%M').md"
  }
}
```

## 2. 项目结构快速浏览指南

### 第一步：了解项目类型
```bash
# 查看项目配置文件
cat app.json
cat project.config.json
```

### 第二步：核心目录结构
```bash
# 1. 查看整体结构
find . -type f -name "*.json" | head -20
ls -la cloudfunctions/ pages/ images/

# 2. 云函数目录（核心业务逻辑）
ls cloudfunctions/
# 重要函数：
# - initDatabase/     # 数据库初始化
# - checkAdmin/       # 权限检查
# - user/register/    # 用户注册
# - user/getProfile/  # 获取用户信息
# - user/updateProfile/ # 更新用户信息

# 3. 页面目录（前端界面）
ls pages/
# 核心页面：
# - index/           # 首页
# - detail/          # 书籍详情
# - register/        # 注册页面
# - profile/         # 个人中心
# - challenge/       # 挑战页面
```

### 第三步：使用AI助手快速探索
在Claude Code中使用以下命令：

```markdown
## 探索项目结构

请帮我：
1. 使用Glob工具查看项目根目录的所有.md文件，了解文档结构
2. 查看cloudfunctions目录下的所有云函数
3. 查看pages目录下的所有页面
4. 读取README.md了解项目概述
5. 读取IMPLEMENTATION_PLAN.md了解实施计划
6. 检查当前git状态和最近提交
```

### 第四步：关键文件识别
| 文件类型 | 关键文件 | 用途 |
|---------|---------|------|
| 配置 | `app.json` | 小程序全局配置 |
| 配置 | `project.config.json` | 项目配置 |
| 文档 | `README.md` | 项目说明 |
| 文档 | `IMPLEMENTATION_PLAN.md` | 7周实施计划 |
| 脚本 | `cloudfunctions/initDatabase/index.js` | 数据库初始化 |
| 核心 | `cloudfunctions/user/register/index.js` | 用户注册逻辑 |

### 第五步：数据库结构了解
```bash
# 查看数据库初始化逻辑
cat cloudfunctions/initDatabase/index.js
```


## 5. 最佳实践

### 会话管理
1. **每次会话专注一个主题** - 避免多个无关任务混杂
2. **及时记录** - 完成每个步骤后更新会话文件
3. **文件引用** - 使用Markdown链接格式：[文件名](路径)
4. **状态跟踪** - 明确标记任务状态（待办/进行中/完成）

### 代码协作
1. **小步提交** - 每个功能点单独提交
2. **描述清晰** - 提交信息说明"为什么"而不仅是"做了什么"
3. **文档更新** - 代码变更时同步更新相关文档
4. **测试优先** - 重要功能添加测试用例

### 安全规范
1. **敏感信息** - 不提交密钥、密码等敏感信息
2. **环境配置** - 使用环境变量管理配置
3. **权限控制** - 遵循最小权限原则
4. **输入验证** - 对所有用户输入进行验证

## 6. 故障排除流程

### 问题诊断步骤
1. **现象描述** - 具体错误信息、操作步骤
2. **环境检查** - 版本、配置、权限
3. **日志分析** - 查看相关日志文件
4. **简化重现** - 最小化重现步骤
5. **假设验证** - 提出假设并验证

### 常见问题
| 问题 | 检查点 | 解决方案 |
|------|--------|----------|
| 云函数部署失败 | 1. 函数名是否正确<br>2. 环境ID是否一致<br>3. 私钥文件是否存在 | 1. 确认函数名<br>2. 统一环境ID<br>3. 检查private.key |
| 数据库连接失败 | 1. 环境配置<br>2. 权限设置<br>3. 网络连通性 | 1. 检查环境ID<br>2. 确认数据库权限<br>3. 测试网络连接 |
| 页面无法显示 | 1. 路径是否正确<br>2. 文件是否存在<br>3. 语法错误 | 1. 检查路由配置<br>2. 确认文件路径<br>3. 查看控制台错误 |
